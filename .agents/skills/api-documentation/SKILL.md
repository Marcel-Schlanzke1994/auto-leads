# api-documentation

## Purpose
Dokumentiert REST-Endpunkte, Felder und Fehlerverhalten.

## When to use
- Bei neuen/angepassten API-Routen.

## Workflow
1. Request/Response-Felder inkl. Typ und Validierungsregeln dokumentieren.
2. Fehlercodes (`4xx/5xx`) mit Beispielpayload beschreiben.
3. Versionierungs-/Breaking-Change-Hinweise ergänzen.

## Safety Guardrails
- Keine unstabilen internen IDs oder Geheimnisse in Beispielen veröffentlichen.

## Output
- Kurze Zusammenfassung, getroffene Annahmen, Risiken und nächste Schritte.
