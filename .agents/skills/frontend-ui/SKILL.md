# frontend-ui

## Purpose
Verbessert Templates/JS/CSS mit Fokus auf Lead-Workflow-Usability.

## When to use
- Bei Dashboard-, Listen- oder Detailseiten-Änderungen.

## Workflow
1. UI-Änderung auf klare Nutzeraufgabe mappen.
2. Zustände (loading/error/empty) sichtbar behandeln.
3. Semantisches HTML und zugängliche Interaktionen sicherstellen.

## Safety Guardrails
- Keine Inline-Secrets oder harte Backend-Endpunkte.

## Output
- Kurze Zusammenfassung, getroffene Annahmen, Risiken und nächste Schritte.
